## 使用指南

### 1. 经历萃取阶段

根据用户提供经历，查找对应能力标签：

```
用户说：我在学生会做过外联部干事

查表：干事级别 → 可证明：执行力、沟通表达、团队协作、学习能力

追问方向：
- 具体做了什么工作？（验证执行力）
- 需要和哪些人沟通？（验证沟通表达）
- 有什么成果？（量化结果）
```

### 2. 经历筛选阶段

根据目标岗位能力需求，筛选匹配经历：

```
目标岗位：管培生
岗位需求：学习能力、沟通表达、团队协作、领导力

用户经历：
1. 学生会主席 → 领导力、组织协调 ✓ 高匹配
2. 销售兼职 → 沟通表达、抗压能力 ✓ 匹配
3. 编程项目 → 问题解决 △ 部分匹配
4. 无关课程作业 → 无相关能力 ✗ 不写入
```

### 3. 简历改写阶段

根据能力标签改写经历描述：

```
原始经历：学生会外联部干事

能力标签：沟通表达、执行力、抗压能力

改写模板：
- 沟通表达：电话沟通企业30家，实地拜访8家
- 执行力：筛选目标企业50家，制定拜访计划
- 抗压能力：在学业压力下坚持完成招商任务
- 结果：成功拉取赞助8000元，覆盖活动成本60%
```

### 4. 能力缺口分析

对照岗位需求找出能力缺口：

```
岗位需求：学习能力、沟通表达、团队协作、领导力、抗压能力

用户已有证据：
✓ 沟通表达：外联经历有证据
✓ 团队协作：社团经历有证据
✓ 执行力：兼职经历有证据
⚠ 领导力：无领导职务经历
⚠ 抗压能力：证据较弱，需追问

缺口建议：
- 领导力：可从项目负责、小组长等经历中挖掘
- 抗压能力：追问兼职期间是否有多任务并行情况
```
