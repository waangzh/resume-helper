# Learning Plan

## 流程

1. 读取匹配诊断中的真实缺口和面试风险。
2. 根据 `JobProfile.role_category` 分类缺口：
   - 技术岗：技术能力缺口、项目经验缺口
   - 通用岗：软技能缺口、经历证据缺口
3. 按优先级拆成短期可补、需要经历证明、暂不建议写入简历三类。
4. 给出 7 天或 14 天行动计划，包含知识点、练习任务、产出物。
5. 推荐资料时优先给官方文档、经典教程或可搜索关键词。

## 缺口分类

### 技术岗

| 类型 | 时间要求 | 处理方式 |
|------|----------|----------|
| 短期可补 | 7-14 天 | 安排学习任务 |
| 需要项目证明 | 14+ 天 | 安排补项目行动 |
| 暂不建议写入简历 | - | 标记为后续目标 |

### 通用岗

| 类型 | 时间要求 | 处理方式 |
|------|----------|----------|
| 短期可补 | 7-14 天 | 安排学习/练习任务 |
| 需要经历证明 | 14+ 天 | 安排补经历行动（参与活动、实习等） |
| 暂不建议写入简历 | - | 标记为后续目标 |

## 软技能提升示例

### 沟通表达

```markdown
### 沟通表达能力提升

**缺口类型：** 短期可补

**提升目标：** 能够清晰、有条理地表达观点

**时间安排：** 7天

**能力提升：**
- 口头表达：练习结构化表达（结论-原因-例证）
- 书面表达：练习写作结构（总-分-总）

**练习任务：**
- 每天录制1分钟观点表达视频，自我复盘
- 阅读《金字塔原理》前3章，完成读书笔记
- 参加一次社团讨论/小组作业，主动发言

**产出物/证据：** 视频记录、读书笔记、发言记录

**推荐资源：**
- 《金字塔原理》（书籍）
- 学校演讲与口才课程（课程）
```

### 团队协作

```markdown
### 团队协作能力提升

**缺口类型：** 需要经历证明

**提升目标：** 能够在团队中有效协作完成任务

**时间安排：** 14天

**能力提升：**
- 协作配合：主动承担团队任务
- 冲突处理：学习冲突解决技巧

**练习任务：**
- 主动参与一个团队项目/课程小组作业
- 在团队中承担具体职责并按时交付
- 记录团队协作中的挑战和解决方式

**产出物/证据：** 项目成果、团队分工记录、协作反思

**推荐资源：**
- 《团队协作的艺术》（书籍）
- 课程小组作业、社团项目（实践机会）
```

### 领导力

```markdown
### 领导力提升

**缺口类型：** 需要经历证明

**提升目标：** 能够带领小团队完成目标

**时间安排：** 14+天

**能力提升：**
- 团队管理：学习任务分配和进度跟进
- 决策判断：学习在不确定情况下做决策

**练习任务：**
- 主动承担社团/班级活动的负责人
- 组织一次小型活动（10人以上）
- 复盘活动过程，总结得失

**产出物/证据：** 活动策划案、活动照片、复盘总结

**推荐资源：**
- 《卓有成效的管理者》（书籍）
- 学生会/社团活动（实践机会）
```

## 输出要求

- 每个短板都要有明确下一步动作
- 学习建议要能反哺简历或面试准备
- 给具体链接时必须确认来源可访问；无法确认时给搜索关键词
- 软技能提升要有具体练习任务和产出物

## 边界

- 不把学习计划写成用户已有能力
- 不推荐过多资料，每个短板 1-3 个高价值资源
- 时间盒默认 7 天（面试前快速补强）或 14 天（补经历证据）
