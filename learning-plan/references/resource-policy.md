# 资料推荐策略

## 资料优先级

| 优先级 | 类型 | 示例 | 说明 |
|--------|------|------|------|
| 1 | 官方文档 | Spring Boot 官方文档 | 权威、最新 |
| 2 | 经典教程 | 廖雪峰 Python 教程 | 系统性强 |
| 3 | 知名课程 | 极客时间、Coursera | 结构化学习 |
| 4 | 搜索关键词 | “Java 并发编程 实战” | 无法确认链接时使用 |

## 链接边界

| 情况 | 处理方式 |
|------|----------|
| 确认可访问 | 写完整 URL |
| 不确定可访问 | 写”搜索关键词：{关键词}” |
| 需要付费/登录 | 标注”需登录”或”付费资源” |

**禁止：** 编造链接、使用过期链接、使用无法验证的第三方链接

## 学习任务要求

每个学习任务必须有**可验证的产出物**：

| 任务类型 | 产出物示例 |
|----------|------------|
| 知识点学习 | 学习笔记、思维导图 |
| 代码练习 | GitHub 仓库、代码片段 |
| 项目补强 | 新增模块、功能演示 |
| 面试准备 | 追问答案要点、模拟回答 |

## 默认时间盒

| 时间盒 | 适用场景 | 内容密度 |
|--------|----------|----------|
| 7 天 | 面试前快速补强 | 2-3 个核心知识点 |
| 14 天 | 补项目证据和系统复盘 | 1 个小项目或功能模块 |

## 学习计划模板

```markdown
### {短板名称}

**缺口类型：** {短期可补/需要项目证明}

**学习目标：** {具体可衡量的目标}

**时间安排：** {天数}天

**每日任务：**
- Day 1-2：{知识点学习}
- Day 3-5：{练习任务}
- Day 6-7：{产出验证}

**推荐资料：**
- {资料1}（官方文档）
- {资料2}（搜索关键词）

**产出物：** {可验证的成果}
```
