# 来源与证据规则

## 证据可信度分级

| 级别 | 说明 | 处理方式 | 示例 |
|------|------|----------|------|
| confirmed | 材料中明确出现 | 直接用于简历 | "GPA 3.8" 写在简历原文中 |
| derived | 可由材料合理归纳 | 保持保守表达 | 项目中使用了 Spring Boot → 可写"了解 Spring Boot" |
| uncertain | 影响质量但无法确认 | 追问或标记待确认 | 写了"熟悉 Redis"但无使用证据 |
| suggested | 只是建议补强 | 不能写成已具备能力 | "建议补充消息队列经验" |

## 证据记录格式

```markdown
| 字段 | 来源 | 原始线索 | 可信度 |
|------|------|----------|--------|
| {技能/经历/成果} | {文件名或链接} | {原句摘要} | {confirmed/derived/uncertain} |
```

## 示例

| 字段 | 来源 | 原始线索 | 可信度 |
|------|------|----------|--------|
| Java 后端开发 | 项目文档.pdf | "使用 Spring Boot 开发商品管理模块" | confirmed |
| Redis 缓存 | 项目文档.pdf | "系统使用 Redis 存储热点数据" | confirmed |
| 高并发经验 | 项目文档.pdf | "系统支持多用户并发访问" | derived |
| 消息队列 | - | 无明确证据 | uncertain |

## 技能可信度判断

| 材料中的表述 | 项目证据 | 可信度判断 |
|--------------|----------|------------|
| 精通/熟练 | 有完整项目实践 | confirmed |
| 熟悉 | 有项目实践 | confirmed |
| 了解/学习过 | 无项目实践 | uncertain |
| 无表述但项目中有使用 | 有代码/配置证据 | derived |

## 数字证据要求

| 数字类型 | 证据要求 | 示例 |
|----------|----------|------|
| GPA/排名 | 简历原文 | "GPA 3.8/4.0" |
| 性能指标 | 项目文档或用户原句 | "响应时间降低 60%" |
| 数据规模 | 项目文档或用户原句 | "日处理 10 万条数据" |
| 接口数量 | 项目文档或用户原句 | "开发 20+ 接口" |

**禁止：** 编造数字、夸大数字、使用无法验证的数字
