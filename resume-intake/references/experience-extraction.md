# 经历萃取指南

> 从非结构化文本、口述草稿、证书照片中提取结构化经历信息。

## 输入类型与处理策略

| 输入类型 | 特点 | 处理策略 |
|----------|------|----------|
| 结构化简历 | 格式规范、信息完整 | 直接提取字段 |
| 非结构化草稿 | 零散、口语化、无格式 | 按模板追问 + 结构化 |
| 口述经历 | 无时间线、缺失细节 | 时间线重构 + STAR 补全 |
| 证书照片 | 需要OCR识别 | 提取关键信息 + 标注来源 |

---

## 非结构化经历草稿处理

### 常见输入形式

```
"我大学干过学生会的，组织过一些活动，还参加过数学建模，拿了个奖。
暑假在奶茶店打过工，还做过家教，教高中数学。"
```

### 萃取流程

1. **识别经历类型** → 社团、竞赛、兼职、家教
2. **提取时间线** → 推断或追问
3. **补全角色信息** → 追问具体职责
4. **追问关键细节** → 用于 STAR 改写
5. **输出结构化记录** → 带"待确认"标记

### 萃取模板

```markdown
| 经历类型 | 时间 | 角色/组织 | 核心职责 | 待追问项 | 证据来源 |
|----------|------|----------|----------|----------|----------|
| 社团 | 待确认 | 学生会 | 组织活动 | 具体职位？活动规模？成果？ | 用户口述 |
| 竞赛 | 待确认 | 数学建模 | 参赛 | 获奖级别？团队角色？负责内容？ | 用户口述 |
| 兼职 | 暑假 | 奶茶店 | 打工 | 具体工作内容？时长？ | 用户口述 |
| 家教 | 待确认 | - | 教高中数学 | 辅导科目？时长？学生进步情况？ | 用户口述 |
```

### 关键追问清单

#### 社团/学生组织

| 追问项 | 目的 |
|--------|------|
| 具体职位是什么？ | 确定角色层级 |
| 团队有多少人？ | 量化规模 |
| 主要负责哪些工作？ | 明确职责 |
| 组织过什么活动？规模多大？ | STAR 的 S 和 T |
| 有什么成果或奖项？ | STAR 的 R |

#### 兼职/实习

| 追问项 | 目的 |
|--------|------|
| 具体做什么工作？ | 明确职责 |
| 工作了多长时间？ | 量化时长 |
| 有什么突出表现？ | STAR 的 R |
| 学到了什么技能？ | 能力提炼 |

#### 竞赛/比赛

| 追问项 | 目的 |
|--------|------|
| 比赛全称是什么？ | 准确命名 |
| 团队几个人？你负责什么？ | 明确角色 |
| 用了什么技术/方法？ | STAR 的 A |
| 获得什么奖项？ | STAR 的 R |

#### 家教/辅导

| 追问项 | 目的 |
|--------|------|
| 辅导什么科目？几年级？ | 明确内容 |
| 辅导了多长时间？ | 量化时长 |
| 学生成绩有提升吗？ | STAR 的 R |
| 有续课或家长好评吗？ | 证明效果 |

## 证书照片识别处理

### 支持的证书类型

| 类型 | 关键提取字段 | 用途 |
|------|-------------|------|
| 获奖证书 | 奖项名称、获奖等级、时间、发证单位 | 竞赛奖项模块 |
| 英语证书 | 证书名称、分数/等级、时间 | 技能模块 |
| 计算机证书 | 证书名称、等级、时间 | 技能模块 |
| 职业资格证书 | 证书名称、等级、时间、发证机构 | 技能/证书模块 |
| 实习证明 | 公司、岗位、时间、工作内容 | 实习经历 |
| 志愿者证书 | 活动、时长、组织方 | 社会实践 |

### OCR 识别要点

1. **优先提取字段：**
   - 证书名称
   - 获奖人姓名（验证身份）
   - 等级/分数
   - 时间
   - 发证单位

2. **识别后处理：**
   ```markdown
   | 字段 | 提取内容 | 置信度 | 原图位置 |
   |------|----------|--------|----------|
   | 证书名称 | 全国大学生数学建模竞赛 | 高 | 标题行 |
   | 获奖等级 | 省级二等奖 | 高 | 正文 |
   | 获奖时间 | 2023年11月 | 高 | 落款处 |
   | 发证单位 | 中国工业 and 应用数学学会 | 中 | 落款处 |
   ```

3. **低置信度处理：**
   - 标记为"待人工确认"
   - 提供原图截图位置
   - 不编造模糊内容

### 证书信息转换为简历内容

**示例：获奖证书**

```markdown
# 原始识别结果
证书名称：蓝桥杯全国软件和信息技术专业人才大赛
获奖等级：省赛一等奖
时间：2024年3月

# 转换为简历竞赛奖项模块
## 竞赛奖项
- 蓝桥杯  省赛一等奖  2024.03
```

**示例：实习证明**

```markdown
# 原始识别结果
公司：XX科技有限公司
岗位：后端开发实习生
时间：2024年6月至2024年9月
工作内容：参与订单系统开发

# 转换为简历实习经历模块（需追问补充细节）
## 实习经历
### XX科技有限公司 / 后端开发组 / 开发实习生  2024.06 - 2024.09

- 参与订单系统开发（待追问：具体负责哪些模块？）
```

### 识别工具调用

当收到证书照片时，使用 AutoClaw 内置图片识别工具：

**使用方式：**

直接用自然语言描述任务，agent 会自动调用图片识别工具执行。

**提示词示例：**

```
使用 autoglm-file-upload 上传本地图片 <图片路径>

使用 autoglm-image-recognition 识别这张证书图片，提取：证书名称、获奖人、等级、时间、发证单位
```

**识别提示词模板：**

```
请识别这张证书照片中的以下信息：
1. 证书名称
2. 获奖人姓名
3. 获奖等级或分数
4. 发证时间
5. 发证单位

如果某项信息不清晰，请标注"不清晰"而非猜测。
```

**无图片识别工具时：**

若图片识别工具不可用，采用交互式追问：
1. 说明无法识别图片内容
2. 请用户口述证书关键信息
3. 不编造或猜测证书内容

## 经历类型识别与归类

### 经历类型判定

| 用户描述关键词 | 归类类型 | 简历模块 |
|----------------|----------|----------|
| 实习、打工、兼职、工作 | 工作经历 | 实习经历 |
| 项目、课程设计、大作业 | 项目经历 | 项目经历 |
| 比赛、竞赛、大赛 | 竞赛经历 | 竞赛奖项 |
| 学生会、社团、班委、团委 | 社团组织 | 项目经历或单独模块 |
| 志愿者、公益、社会实践 | 社会实践 | 竞赛奖项或单独模块 |
| 家教、辅导 | 兼职经历 | 实习经历（视岗位相关性） |

| 经历类型 | 技术岗价值 | 管培生价值 | 通用岗位价值 |
|----------|-----------|-----------|-------------|
| 技术项目 | 高 | 低 | 中 |
| 互联网实习 | 高 | 高 | 高 |
| 学生会主席 | 低 | 高 | 高 |
| 销售兼职 | 低 | 高 | 中 |
| 志愿者活动 | 低 | 中 | 中 |
| 专业竞赛 | 高 | 中 | 高 |

## 最小追问原则

遵循 `minimal-questioning` skill 的原则：

1. **只追问影响简历质量的关键信息**，不追问细节
2. **最多追问 1-3 个问题**
3. **用户未回答时，用"待确认"标记继续生成草案**
4. **提供追问的背景说明**，让用户理解为什么需要这个信息

### 追问示例

```
为了更好地在简历中呈现你的学生会经历，请补充以下信息：
1. 你在学生会担任的具体职位是什么？（干事/部长/副主席等）
2. 你组织/参与的最成功的活动是什么？规模有多大？

如果暂时想不起来细节，我可以先生成一份初稿，你之后再补充。
```
