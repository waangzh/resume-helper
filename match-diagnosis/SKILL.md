# Match Diagnosis

## 流程

1. 对照岗位画像检查简历中已体现的核心能力。
2. 根据 `JobProfile.role_category` 选择评估维度：
   - 技术岗：评估技术能力、项目成果
   - 通用岗：评估软技能、组织经历、活动成果
3. 区分表达不足和真实缺失：前者可改写，后者需要学习或补经历。
4. 标记面试风险项：写入简历但证据弱、容易被追问穿透的内容。
5. 输出匹配诊断，供面试题和学习计划使用。

## 输出结构

### 技术岗

| 类别 | 说明 | 下一步 |
|------|------|--------|
| 已命中能力 | 岗位核心技能有明确证据 | 突出表达 |
| 待优化表达 | 经历存在但缺少技术动作/结果 | 改写 bullet |
| 真实能力缺口 | 无证据，不能写入简历 | 学习/补项目 |
| 面试风险 | 证据弱/范围过大/可能讲不清 | 准备追问答案 |

### 通用岗

| 类别 | 说明 | 下一步 |
|------|------|--------|
| 已命中能力 | 岗位核心能力有明确证据 | 突出表达 |
| 待优化表达 | 经历存在但缺少职责/成果描述 | 用 STAR 改写 |
| 真实能力缺口 | 无证据支撑的能力 | 补经历/弱化表述 |
| 面试风险 | 能力声明但证据不足 | 准备行为面试答案 |

## 评估方式

- 不用虚假百分比分数制造精确感
- 使用档位：高/中/低 或 1-5 档
- 每项建议必须可执行

## 可执行建议示例

### 技术岗

| 问题 | 建议 |
|------|------|
| 缺少量化结果 | 补充接口数量、数据规模、性能指标 |
| 技术描述空泛 | 改为”基于 {技术} 实现 {功能}，结果 {指标}” |
| 证据不足 | 准备追问时的回答要点，或弱化表述 |

### 通用岗

| 问题 | 建议 |
|------|------|
| 缺少量化成果 | 补充活动参与人数、服务时长、转化数据 |
| 能力描述空泛 | 用 STAR 原则描述：情景、任务、行动、结果 |
| 软技能无证据 | 从其他经历中挖掘相关证据，或弱化表述 |
| 领导力证据弱 | 补充带团队人数、完成目标、培养他人等细节 |
