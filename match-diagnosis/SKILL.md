# Match Diagnosis

## 流程

1. 对照岗位画像检查简历中已体现的核心能力。
2. 区分表达不足和真实缺失：前者可改写，后者需要学习或补项目。
3. 标记面试风险项：写入简历但证据弱、容易被追问穿透的内容。
4. 输出匹配诊断，供面试题和学习计划使用。

## 输出结构

| 类别 | 说明 | 下一步 |
|------|------|--------|
| 已命中能力 | 岗位核心技能有明确证据 | 突出表达 |
| 待优化表达 | 经历存在但缺少技术动作/结果 | 改写 bullet |
| 真实能力缺口 | 无证据，不能写入简历 | 学习/补项目 |
| 面试风险 | 证据弱/范围过大/可能讲不清 | 准备追问答案 |

## 评估方式

- 不用虚假百分比分数制造精确感
- 使用档位：高/中/低 或 1-5 档
- 每项建议必须可执行

## 可执行建议示例

| 问题 | 建议 |
|------|------|
| 缺少量化结果 | 补充接口数量、数据规模、性能指标 |
| 技术描述空泛 | 改为”基于 {技术} 实现 {功能}，结果 {指标}” |
| 证据不足 | 准备追问时的回答要点，或弱化表述 |
