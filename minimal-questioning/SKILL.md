# Minimal Questioning

## 流程

1. 读取当前目标、候选人资料和待确认项。
2. 判断哪些缺口会阻塞简历质量、岗位推荐或真实性。
3. 只输出 1-3 个问题，按影响程度排序。
4. 每个问题给出推荐默认值或可选项，降低用户输入成本。

## 追问优先级

1. 岗位类型：实习、校招、社招。
2. 城市、远程、薪资或行业偏好。
3. 最能代表岗位能力的项目。
4. 项目中个人职责、技术栈、成果数字。
5. 不能从材料确认但会写入简历的信息。

## 边界

- 不问已经能从资料确认的问题。
- 不用长表单。
- 用户未回答时继续生成草案，并把相关内容标为“待确认”。
