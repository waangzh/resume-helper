# Minimal Questioning

## 流程

1. 读取当前目标、候选人资料和待确认项。
2. 判断哪些缺口会阻塞简历质量、岗位推荐或真实性。
3. 只输出 1-3 个问题，按影响程度排序。
4. 每个问题给出推荐默认值或可选项，降低用户输入成本。

## 追问优先级

### 技术岗

| 优先级 | 问题类型 | 示例 |
|--------|----------|------|
| 1 | 岗位类型 | 实习、校招、社招？ |
| 2 | 城市/薪资偏好 | 北京/上海/杭州？ |
| 3 | 代表项目 | 哪个项目最能体现岗位能力？ |
| 4 | 职责细节 | 项目中你负责哪些模块？ |
| 5 | 成果数字 | 接口数量、用户规模、性能指标？ |

### 通用岗

| 优先级 | 问题类型 | 示例 |
|--------|----------|------|
| 1 | 岗位类型 | 实习、校招、社招？ |
| 2 | 城市/薪资偏好 | 北京/上海/杭州？ |
| 3 | 代表经历 | 哪段经历最能体现岗位能力？（学生会/实习/兼职） |
| 4 | 职责成果 | 在学生会担任什么职务？组织过什么活动？ |
| 5 | 量化结果 | 活动参与人数？拉取赞助金额？服务时长？ |

## 问题格式

```markdown
{问题}？

选项：
- A: {选项A}（推荐）
- B: {选项B}
- C: {选项C}

原因：{为什么需要这个答案}
```

## 边界

- 不问已经能从资料确认的问题
- 不用长表单，最多 3 个问题
- 用户未回答时继续生成草案，相关内容标为”待确认”
