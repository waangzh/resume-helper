# 追问策略

## 问题设计原则

| 原则 | 说明 |
|------|------|
| 只问一个决策点 | 不复合多个问题 |
| 提供可选项 | 降低用户输入成本 |
| 说明原因 | 让用户理解为什么需要这个答案 |

## 问题格式模板

```markdown
{问题}？

选项：
- A: {选项A}（推荐：{推荐理由}）
- B: {选项B}
- C: {选项C}

影响：{这个答案会影响什么}
```

## 示例

### 岗位类型追问

```markdown
你这次更偏向实习岗还是校招正式岗？

选项：
- A: 实习岗（推荐：实习经历可作为加分项）
- B: 校招正式岗
- C: 都可以

影响：这会影响简历标题、岗位关键词和推荐岗位范围
```

### 城市偏好追问

```markdown
你接受哪些城市？

选项：
- A: 北京、上海、杭州、深圳（推荐：互联网岗位较多）
- B: 只接受特定城市
- C: 不限城市

影响：如果不确定，我先按一线城市生成推荐清单
```

### 项目选择追问

```markdown
哪个项目最能体现目标岗位能力？

选项：
- A: {项目1}
- B: {项目2}
- C: 都不够强

影响：我会把它放在简历前部并重点准备面试追问
```

## 追问优先级

| 优先级 | 问题类型 | 影响 |
|--------|----------|------|
| 1 | 岗位类型 | 简历标题、关键词、推荐范围 |
| 2 | 城市/薪资偏好 | 推荐岗位筛选 |
| 3 | 代表项目选择 | 简历结构、面试重点 |
| 4 | 职责细节 | Bullet 改写质量 |
| 5 | 成果数字 | 量化表达可信度 |

## 边界

- 不问已经能从资料确认的问题
- 最多 3 个问题
- 用户未回答时继续生成草案，相关内容标为”待确认”
