# 简历 HTML 版式质量约束

本文件用于约束 Markdown 到 HTML 预览的生成质量。目标是让默认 HTML 接近参考简历的阅读节奏：单栏 A4、蓝色标题、细分隔线、正文不拥挤、照片位不压正文。

## 内容结构约束

- Markdown 只使用一级标题、二级标题、三级标题、普通段落和一级 bullet，不使用表格、多栏、嵌套列表或图片化文字。
- 一级标题只放姓名；首个 `##` 标题之前的普通行都作为基本信息区，生成时必须与照片组合成同一个页眉整体。
- 页眉文字区第一行是姓名，第二行固定为求职意向摘要，格式为 `求职意向：{岗位} | {城市} | {薪资} | {到岗时间}`，分隔符使用竖线。
- 页眉基础信息使用两列三行，顺序严格为：`年龄  性别`、`籍贯  工作年限`、`电话  邮箱`；字段之间使用两个以上空格或 Tab 分隔。
- 基础信息项建议使用 `标签：内容` 格式，例如 `电话：15588888881  邮箱：qmjianli@qq.com`。
- 二级标题用于标准模块：教育背景、技术技能、项目经历、实习经历、竞赛奖项、自我评价。
- 教育背景固定使用 3 行：第一行写 `**{学校}**  **{专业}（{学历}）**{起止时间}  `，第二行写 `专业成绩：...`（与专业排名之间用多个空格符分割，但是保持在同一行），第三行写 `主修课程：...`。
- 教育背景第一行渲染为左中右三段：左侧学校、中间专业和学历、右侧时间；成绩行的“专业成绩”加粗；课程行的“主修课程”加粗。
- 主修课程只保留 4-6 门与目标岗位相关的课程，避免把全部课程堆入简历。
- 项目或经历标题使用三级标题，格式建议为 `项目名  项目类型  起止时间`（多个空格符号分割，项目类型位于整行的中间位置），不要把多段说明塞进标题。
- 每个项目正文先用 1 段说明技术栈和业务目标，再放 2-3条 bullet；单条 bullet 控制在 1-2 行。
- 技能模块按类别写短句，不写长段落，不使用技能进度条。

## HTML 视觉约束

- 页面固定 A4 宽度，默认单栏；正文区不得被照片、工具栏或装饰元素遮挡。
- 页面左右内边距保持在 16-18mm；页眉、模块标题、正文段落、列表和教育背景等所有 section 的左右边界必须与页面内容区一致，不得单独使用负 margin 或额外缩进造成页边距不一致。
- 正文字号不低于 13px，默认行高不低于 1.6；模块标题字号应明显大于正文。
- 模块之间必须有稳定间距，二级标题下方必须有分隔线或浅色标题带。
- bullet 行距必须大于普通段落的最小行距，避免连续列表挤在一起。
- 基本信息页眉必须整体参考用户给定图片：左侧照片，右侧依次为姓名、求职意向摘要、两列三行基础信息；照片和右侧文字作为同一布局组件参与文档流，后续“教育背景”不得进入照片高度范围。
- 页眉左侧为照片区域，宽 98px、高 126px。照片左边界必须与页面内容左边界对齐，不放额外说明文字。
- 姓名、求职意向摘要和基础信息位于照片右侧并左对齐；照片与文字区水平间距约 50px。
- 求职意向摘要和基础信息文字必须继承正文默认字号和行高，不单独设置更小字号或更小行高；姓名字号明显大于正文，字距可适度拉开以贴近参考图。
- 基础信息区使用两列三行网格和稳定行距，不把所有信息挤成一行；每项按“图标 / 标签 / 内容”三段排列，两字字段显示为接近四字宽度，例如 `年    龄`、`电    话`。
- 装饰色只用于姓名、模块标题和分隔线；正文保持深色普通文本。

## 生成与验收约束

- 默认先生成 HTML 预览，PDF 仅从最终 HTML 打印生成，不维护第二套 PDF 内容。
- HTML 预览提供正文文字编辑功能，但不提供拖动头像、移动 section、调整页边距等布局微调功能。
- 生成 HTML 后应检查页面是否存在文本重叠、照片遮挡、标题贴边、列表过密、工具栏进入打印区域。
- 打印样式必须隐藏工具栏，只保留简历正文。
- 打印样式必须使用 `print-color-adjust: exact` 和 `-webkit-print-color-adjust: exact`，确保模块标题背景色、浅色标题带和装饰性分隔元素进入 PDF。
- 如果内容无法在一页内清晰排下，优先压缩内容和 bullet 数量，不通过降低字号或行高强行塞满。
