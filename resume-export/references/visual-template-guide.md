# 参考图视觉模板指南

参考图位于 `assets/reference-images/`，仅作为视觉依据；默认生成应使用结构化 HTML/CSS 样式规则，不直接依赖图片 OCR。

HTML/CSS 的实际可用样式集中在 `assets/templates/resume-web/resume.css`。不要从历史行政简历或截图中复制示例个人信息。

## 共同可取规则

- 单栏 A4 简历，模块从上到下排列，阅读顺序稳定，一页优先。
- 顶部基本信息严格参考用户给定图片：左侧照片，右侧展示姓名、求职意向（只显示岗位）和两列三行联系方式。
- 模块标题醒目，使用横线或浅色背景分隔。
- 正文以短句 bullet 为主，项目/实习每项 2-3 条，减少大段自我评价。
- 版面留白克制，颜色只用于标题和分隔，不影响可读性；不要为凑满页面扩写内容。

## 需要改造的地方

- 页眉可沿用参考图的照片 + 文字组合、字段分割和两列三行网格；不沿用婚姻、技能进度条等与投递无关的信息。
- 技术岗应突出技能栈、项目经历、实习/竞赛，而不是自我评价和技能进度条。
- 技能进度条不利于 ATS 和真实性表达，默认改为分组技能文本。
- 课程和证书只保留与目标岗位相关或能增强可信度的内容。

## 风格映射

### classic-blue

- 来源：`classic-blue.png`。
- 特征：蓝色姓名标题、顶部信息两列、模块标题蓝色、细横线分隔。
- 使用场景：通用投递、校招、希望简洁但有视觉识别。

### teal-ribbon

- 来源：`teal-ribbon.png`。
- 特征：深青矩形模块标签、横线分区、版面紧凑；标题块不使用斜切、三角缺口或装饰尾巴。
- 使用场景：内容较多但仍需一页排下；应同时参考它的窄页边距、短句密度和紧凑分区。

### light-blue-band

- 来源：`light-blue-band.png`。
- 特征：浅蓝横条模块标题、信息区清晰、分区感强。
- 使用场景：希望模块更醒目，或打印/屏幕阅读优先。

### emerald-clean

- 来源：无独立参考图，复用当前单栏 HTML 结构。
- 特征：绿色强调色、浅绿色模块底、边线克制，阅读感清爽。
- 使用场景：通用技术岗、产品/运营类岗位，适合希望简洁但不偏蓝色的版本。

### burgundy-formal

- 来源：无独立参考图，复用当前单栏 HTML 结构。
- 特征：酒红强调色、细线分隔、标题更正式。
- 使用场景：金融、咨询、管理培训生、偏正式企业投递。

### graphite-amber

- 来源：无独立参考图，复用当前单栏 HTML 结构。
- 特征：石墨灰正文层级配琥珀色分隔，稳重但有识别度。
- 使用场景：中高级技术岗、工程/数据类岗位、希望弱化蓝色模板感的版本。

### minimal-gray

- 来源：无独立参考图，复用当前单栏 HTML 结构。
- 特征：黑白灰为主，少量线条分隔，ATS 友好。
- 使用场景：平台解析风险较高、需要极简投递版本，或用户明确要求低装饰。

## 默认选择

- 技术岗默认：`classic-blue`。
- ATS 优先或平台解析风险高：`plain`。
- 用户只要求换一种颜色但不改结构：优先在 `classic-blue`、`emerald-clean`、`burgundy-formal`、`graphite-amber`、`minimal-gray` 中选择。
- 用户明确说“参考第一张”：`teal-ribbon`。
- 用户明确说“参考第二张”：`classic-blue`。
- 用户明确说“参考第三张”：`light-blue-band`。
