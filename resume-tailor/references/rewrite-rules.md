# 简历改写规则

优先句式：

- “基于 {技术} 实现 {功能/模块}，用于 {业务场景}。”
- “设计并完成 {接口/页面/测试/模型}，提升 {效率/稳定性/可维护性}。”
- “结合 {数据库/缓存/框架} 处理 {数据/流程/问题}。”
- “针对 {问题/需求}，通过 {方案/动作} 完成 {结果/产出}。”
- “在 {项目/团队} 中承担 {个人职责}，交付 {模块/文档/测试/上线结果}。”

禁止句式：

- “精通/深入掌握”除非资料中有强证据。
- “大幅提升/显著优化”但没有指标或具体动作。
- “独立负责全部系统”但资料只显示参与。
- “熟练使用/精通”但只有课程学习或无项目证据。
- “提升 XX%”但没有来源、口径或待确认说明。

待确认写法：

- “可补充：接口数量、数据规模、性能指标或上线情况。”
- “待确认：该项目中是否使用 Redis 缓存或消息队列。”

Bullet 自检：

- 是否说明个人动作，而不是团队整体成果。
- 是否包含岗位关键词，但不过度堆砌。
- 是否能被面试追问时讲清楚。
- 是否有数字、范围、结果；没有则用具体交付物替代。
