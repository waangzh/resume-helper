# 计算机专业校招简历编写规范

## 总体原则

- 首要目标是生成或优化可投递简历，不是完整记录经历；所有内容围绕目标岗位取舍。
- 应届生、实习生默认控制在 1 页以内；内容特别多时先删减低相关经历，不通过压缩字号硬塞。
- 表达要简洁、正式、易读，优先高质量信息密度；避免花哨图标、大面积色块、复杂表格、技能进度条和图片化文字。
- 所有能力、技术和成果必须能从用户资料或公开 JD 找到依据；无依据内容只能写入待确认项，不能写进最终简历正文。

## 推荐结构

默认模块顺序如下，按岗位相关性微调：

1. 基本信息
2. 教育背景
3. 技术技能
4. 项目经历
5. 实习经历（若有）
6. 竞赛奖项
7. 自我评价

如果实习经历明显强于项目经历，可把实习经历放在项目经历之前。不要默认单独设置“求职意向”模块；目标岗位可体现在文件名、标题旁短语或基本信息中的一行。

## 模块规则

### 基本信息

- 建议只包含姓名、手机号、常用邮箱、求职意向、Github、博客。
- 不写身份证号、家庭住址、身高体重、婚姻状况、政治面貌等技术岗无关信息，除非用户或岗位明确要求。
- 照片是必须项；HTML 模板保留上传照片位置，但简历正文不编造照片或外貌信息。

### 教育背景

- 写学校、学院/专业、学历、预计毕业时间。
- GPA、专业排名、奖学金、校级荣誉只在有优势且有依据时写。
- 相关课程可选，只保留与岗位密切相关的 3-6 门；不要罗列大量课程。
- 格式严格参考示例，不同类的换行书写，同行之间不同部分用空格间隔，保证排版不拥挤

例如：

> 湖南大学        		计算机科学与技术    	本科    	2019.09 - 2023.06
> GPA：3.8 / 4.0						专业排名：前 10%
> 相关课程：数据结构、操作系统、计算机网络、数据库系统、编译原理

### 技术技能

- 只写2-3行，内容一定不要过多
- 只写自己真正掌握、可以在面试中展开说明的技术；不为了全面堆砌关键词。
- 不写“学习能力强、责任心强、沟通能力好”等软性描述。
- 避免“精通”等绝对词，除非用户资料中有强证据。

例如：

> - 编程语言：熟悉 Java，了解 Python，掌握常用集合、并发编程、JVM 基础。
> - 后端框架：熟悉 Spring Boot、Spring MVC、MyBatis，了解 Spring Cloud 常用组件。
> - 数据库：熟悉 MySQL 索引、事务、锁机制和 SQL 优化，了解 Redis 常见数据结构、缓存穿透、缓存击穿、缓存雪崩解决方案。
> - 中间件：了解 RabbitMQ / Kafka 的基本使用场景，能够基于消息队列实现异步处理和削峰。
> - 计算机基础：掌握数据结构、操作系统、计算机网络、数据库等基础知识。
> - 工程工具：熟悉 Git、Maven、Docker、Linux 常用命令。

### 项目经历

- 选择 2-3 个最能证明岗位匹配度的项目，不求数量多。
- 每个项目包含项目名称、时间、可选技术栈/角色、1 句项目简介和 2-4 条个人职责与成果（一定不要超过4条）。
- Bullet 使用“动作 + 技术/任务 + 结果”表达，突出个人贡献、解决的问题和交付成果。
- 能量化时写真实数字；没有数字时写模块、接口、功能覆盖、问题定位过程或交付物。
- 多人项目必须突出本人负责部分，不写成团队整体介绍。

### 实习经历

- 若有实习，写公司/部门/岗位、时间、2-4 条工作内容与成果。
- 避免“参与开发、协助维护”这类空泛职责；写清实际任务、使用技术、结果价值。
- 只保留与目标岗位最相关的内容，不把所有工作流水账堆上去。

### 竞赛奖项

- 保留专业竞赛、奖学金、校级及以上荣誉、科研立项、论文、专利等有价值内容。
- 使用紧凑格式：`奖项名称｜级别 / 名次｜时间`。
- 默认 2-4 行；与岗位关系弱的奖项合并或删除。

### 自我评价

- 控制在 2-3 行内。
- 只概括求职方向、核心技术能力和个人优势；禁止“性格开朗、积极向上、吃苦耐劳”等空话。

例如：

> - 具备完整后端项目开发经验，能够独立完成接口设计、数据库设计、开发、测试和部署。
> - 注重代码质量和文档沉淀，项目中曾编写接口文档、部署文档和问题排查记录。
> - 学习能力较强，能够根据项目需要快速上手 Redis、RabbitMQ、Docker 等技术。

## 篇幅控制

- 基本信息：1-2 行。
- 教育背景：3-4 行。
- 技术技能：2-4 行。
- 项目经历：主要篇幅，2-3 个项目，每个 2-4 条 bullet。
- 实习经历：若有，1-2 段，每段 2-4 条 bullet。
- 竞赛奖项：2-4 行。
- 自我评价：可省略，最多 2-3 行。

删减优先级：无关课程、弱相关奖项、空泛自评、低价值项目、重复技术词。

## 常见避坑

- 不堆砌无法解释原理或项目应用细节的技术栈。
- 不只写项目功能，要写个人贡献。
- 不写“提升效率、优化性能、增强稳定性”而没有动作、场景或证据。
- 不把学习计划、待补能力、建议项目写成已完成经历。
- 不让前端、社团、课程作业等弱相关内容抢占与目标岗位更相关的项目位置。

## ATS 与投递兼容

- 最终交付以可编辑 HTML 为主；PDF 仅用户明确要求时由 HTML 打印导出。
- 联系方式使用普通文本，不能放进图片、页眉图或不可解析文本框。
- 模块标题、项目名、学校名、公司名和 bullet 必须是可编辑 DOM 文本。
- 岗位关键词自然出现在技术技能和项目经历中，不做无意义重复。
