# 计算机专业校招简历编写规范

## 总体原则

- 首要目标是生成或优化可投递简历，不是完整记录经历；所有内容围绕目标岗位取舍。
- 应届生、实习生默认控制在 1 页以内；内容特别多时先删减低相关经历，不通过压缩字号硬塞。
- 表达要简洁、正式、易读，优先高质量信息密度；避免花哨图标、大面积色块、复杂表格、技能进度条和图片化文字。
- 所有能力、技术和成果必须能从用户资料或公开 JD 找到依据；无依据内容只能写入待确认项，不能写进最终简历正文。

---

## Markdown 输出格式规范（必须严格遵守）

> 本规范是简历 Markdown 输出的唯一格式契约，解析脚本和 CSS 渲染均依赖此格式。格式错误会导致 HTML 预览排版混乱。

### 分隔符约定

| 用途 | 分隔符 | 示例 |
|------|--------|------|
| 基本信息、教育背景字段间 | **两个以上空格** 或 **Tab** | `年龄：22  性别：男` |
| 求职意向多项间 | 竖线 + 前后空格 | `求职意向：Java后端 \| 北京 \| 随时到岗` |
| 竞赛奖项字段间 | 两个以上空格 | `蓝桥杯  省赛一等奖  2024.03` |

### 一级标题（姓名）

```markdown
# 张三
```

约束：
- 只写姓名，不附加其他信息
- 姓名前不加修饰词

### 求职意向行

```markdown
求职意向：Java后端实习
```

约束：
- 固定格式：`求职意向：{岗位}`
- 只显示求职岗位，不包含城市、到岗时间等其他信息

### 基本信息区

```markdown
年龄：22  性别：男
籍贯：湖南  工作年限：无
电话：13800138000  邮箱：example@email.com
```

约束：
- 每行两项，使用 **两个以上空格** 分隔
- 固定顺序：第一行年龄/性别，第二行籍贯/工作年限，第三行电话/邮箱
- 两字标签无需额外空格，解析时会自动展开为四字宽度
- 缺失字段写”待确认”，不要留空

### 教育背景模块

```markdown
## 教育背景

**湖南大学**  **计算机科学与技术（本科）**  2021.09 - 2025.06
专业成绩：GPA 3.8/4.0  专业排名：前10%
主修课程：数据结构、操作系统、计算机网络、数据库系统
```

约束：
- 第一行：学校、专业（学历）、时间，使用 **两个以上空格** 分隔
- 学校和专业必须用 `**` 加粗
- 学历写在专业后的括号内
- 渲染后三列全部加粗显示：**时间**、**学校**、**专业（学历）**
- 第二行：专业成绩和排名，使用 **两个以上空格** 分隔
- 第三行：主修课程，用中文顿号分隔，保留 4-6 门
- 三行结构完整，不可增减行数

### 技术技能模块

```markdown
## 技术技能

- 编程语言：熟悉 Java，掌握常用集合、并发编程基础
- 后端框架：熟悉 Spring Boot、MyBatis，了解 Spring Cloud
- 数据库：熟悉 MySQL 索引、事务，了解 Redis 缓存方案
- 工具：Git、Maven、Docker、Linux 常用命令
```

约束：
- 每项以 `- ` 开头（短横线 + 空格）
- 格式：`- {类别}：{技能描述}`
- 类别后使用中文冒号
- 控制在 2-4 行，不要过多

### 项目经历模块

```markdown
## 项目经历

### 电商后台管理系统  2023.09 - 2024.01

基于 Spring Boot + Vue 的 B 端管理系统，支持商品管理、订单处理、数据统计。

- 负责商品模块后端开发，设计并实现 SKU 管理、库存同步接口
- 使用 Redis 缓存热点数据，接口响应时间降低 60%
- 编写单元测试 50+ 用例，核心模块覆盖率达 85%

### 另一个项目名  中间字段（可选）  2024.03 - 2024.06

一句话项目简介。

- bullet 1
- bullet 2
```

约束：
- 项目标题使用 `### ` 三级标题
- 格式：`### {项目名称}  {时间}` 或 `### {项目名称}  {中间字段}  {时间}`
- 名称、中间字段、时间之间用 **两个以上空格** 分隔
- 渲染后：项目名称左对齐，时间右对齐，中间字段居中
- 时间统一格式：`YYYY.MM - YYYY.MM` 或 `YYYY.MM - 至今`
- 项目简介在标题后第一个段落，不用 bullet
- 每个 bullet 以 `- ` 开头
- 每个项目 2-3 条 bullet，不要超过 3 条
- **数量边界：** 项目数量以用户提供的材料为准，不自动补充；只有用户主动提出要求时才可额外添加

### 实习经历模块

```markdown
## 实习经历

### 某科技公司 / 后端开发组 / 开发实习生  2024.06 - 2024.09

- 参与订单系统重构，负责物流跟踪模块接口开发与测试
- 协助排查线上问题 10+ 次，产出问题分析文档 3 份
```

约束：
- 标题格式：`### {公司} / {部门} / {岗位}  {时间}`
- 公司、部门、岗位之间使用 ` / ` 分隔（斜线前后各一个空格）
- 渲染后：公司左对齐，时间右对齐，部门/岗位作为中间字段居中
- 没有 bullet 形式的项目简介，直接列出 bullet
- 每段实习 2-3 条 bullet

### 竞赛奖项模块

```markdown
## 竞赛奖项

- 蓝桥杯  省赛一等奖  2024.03
- 数学建模国赛  国家级二等奖  2023.11
```

约束：
- 每项以 `- ` 开头
- 格式：`- {奖项名称}  {获奖等级}  {时间}`
- 字段间使用 **两个以上空格** 分隔
- 时间格式：`YYYY.MM`
- 渲染后：奖项名称左对齐，时间右对齐，获奖等级居中
- 三列全部加粗显示
- 控制 2-4 行

### 自我评价模块（可选）

```markdown
## 自我评价

具备完整的后端项目开发经验，能够独立完成模块设计、开发与测试。学习能力强，可快速上手新技术。
```

约束：
- 不使用 bullet，直接写 1-2 段短文
- 每段控制在 1-2 行
- 最多 2-3 行，可省略此模块

## 推荐结构

默认模块顺序如下，按岗位相关性微调：

1. 基本信息
2. 教育背景
3. 技术技能
4. 项目经历
5. 实习经历（若有）
6. 竞赛奖项
7. 自我评价（可选）

如果实习经历明显强于项目经历，可把实习经历放在项目经历之前。

### 内容选择原则

- 按参考图结构准备基本信息：姓名、求职意向（只显示岗位）、年龄、性别、籍贯、工作年限、手机号、常用邮箱。没有证据的字段写入待确认，不编造。
- 不写身份证号、家庭住址、身高体重、婚姻状况、政治面貌等技术岗无关信息，除非用户或岗位明确要求。
- 照片是基本信息整体的一部分；HTML 模板保留左侧上传照片位置，右侧排姓名、求职意向和两列三行基础信息，但简历正文不编造照片或外貌信息。

## 篇幅控制

- 基本信息：页眉区固定为左侧照片 + 右侧文字，不按普通正文行数统计。
- 教育背景：3-4 行。
- 技术技能：2-4 行。
- 项目经历：主要篇幅，2-3 个项目，每个 2-3 条 bullet。
- 实习经历：若有，1-2 段，每段 2-3 条 bullet。
- 竞赛奖项：2-4 行。
- 自我评价：可省略，最多 2-3 行。

删减优先级：无关课程、弱相关奖项、空泛自评、低价值项目、重复技术词。

## 常见避坑

- 不堆砌无法解释原理或项目应用细节的技术栈。
- 不只写项目功能，要写个人贡献。
- 不写“提升效率、优化性能、增强稳定性”而没有动作、场景或证据。
- 不把学习计划、待补能力、建议项目写成已完成经历。
- 不让前端、社团、课程作业等弱相关内容抢占与目标岗位更相关的项目位置。

## ATS 与投递兼容

- 最终交付以可编辑 HTML 为主；PDF 仅用户明确要求时由 HTML 打印导出。
- 联系方式使用普通文本，不能放进图片、页眉图或不可解析文本框。
- 模块标题、项目名、学校名、公司名和 bullet 必须是可编辑 DOM 文本。
- 岗位关键词自然出现在技术技能和项目经历中，不做无意义重复。
