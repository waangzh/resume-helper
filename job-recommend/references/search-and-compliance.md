# 岗位搜索与合规

## 按岗位类型搜索

### 技术岗搜索策略

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1 | 岗位名 + 类型 | "Java后端 实习" |
| 2 | 加入城市 | "Java后端 实习 北京" |
| 3 | 加入核心技术 | "Java后端 实习 北京 Spring" |
| 4 | 指定平台 | "Java后端 实习 北京 Spring BOSS直聘" |

### 通用岗搜索策略

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1 | 岗位名 + 变体名 | "管培生 OR 管理培训生 OR 储备干部" |
| 2 | 加入类型 + 城市 | "管培生 校招 北京" |
| 3 | 加入行业偏好 | "管培生 校招 北京 互联网" |
| 4 | 指定平台 | "管培生 校招 北京 BOSS直聘" |

**通用岗岗位名称变体：**

| 主岗位名 | 变体名称 |
|----------|----------|
| 管培生 | 管理培训生、储备干部、管培、MT |
| 产品运营 | 运营、用户运营、内容运营、活动运营 |
| 人力资源 | HR、人事、招聘专员、HR助理、HRBP |
| 市场销售 | 销售代表、商务拓展、BD、市场专员 |
| 行政助理 | 行政专员、文员、前台、行政助理 |

## 浏览器操作工具

### 推荐工具（优先使用）

| 工具 | 功能 | 调用方式 | 适用场景 |
|------|------|----------|----------|
| `autoglm-browser-agent` | 智能浏览器自动化 | `autoclaw task=”...”` | 岗位搜索、打开页面、表单填写、投递操作 |
| `autoglm-websearch` | 网络搜索 | `python websearch.py “关键词”` | 岗位关键词搜索 |
| `autoglm-open-link` | 打开链接获取内容 | `python open-link.py “URL”` | 读取岗位详情页正文 |

**autoglm-browser-agent 使用要点：**
- 任务描述必须包含明确数量（默认5）
- 需要登录时会让用户手动操作，然后继续
- 每轮对话只调用一次，等待结果返回
- 详细使用说明见该 skill 的 SKILL.md

### 替代工具（无 autoglm 时使用）

| 工具 | 功能 | 适用场景 |
|------|------|----------|
| `mcp__tavily__tavily_search` | 网页搜索 | 岗位关键词搜索 |
| `WebSearch` | 网页搜索 | 岗位关键词搜索 |
| `WebFetch` | 获取网页内容 | 读取岗位详情页 |
| `mcp__chrome-devtools__*` | Chrome DevTools 操作 | 浏览器自动化、表单填写 |
| `mcp__plugin_playwright_playwright__*` | Playwright 浏览器操作 | 浏览器自动化、投递流程 |
| `agent-browser` skill | 浏览器自动化 | 表单填写、投递操作 |

### 无浏览器工具时

若以上工具均不可用：
1. 输出搜索关键词和筛选规则
2. 提供平台入口链接
3. 生成待用户自行确认的岗位清单模板
4. 不编造岗位或链接

## 搜索策略

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 组合关键词 | 岗位名 + 实习/校招 + 城市 + 核心技术 |
| 2 | 选择平台 | 优先用户指定平台，否则使用公开可访问渠道 |
| 3 | 筛选结果 | 只保留有明确岗位信息的详情页 |
| 4 | 记录信息 | 标题、公司、地点、平台、链接、检索日期 |

## 支持平台

| 平台 | 类型 | 注意事项 |
|------|------|----------|
| BOSS 直聘 | 综合招聘 | 可能需要登录，遇到验证码停下 |
| 拉勾 | 互联网 | 可能需要登录 |
| 猎聘 | 中高端 | 可能需要登录 |
| 智联招聘 | 综合招聘 | - |
| 前程无忧 | 综合招聘 | - |
| 牛客 | 校招/实习 | - |
| 实习僧 | 实习 | - |

| 公司 ATS | 官方直投 | 适合精准投递 |

## 链接要求

**必须：** 岗位详情页链接，能直接打开到具体岗位

**禁止：**
- 公司官网首页
- 招聘首页
- 搜索结果页
- 聚合页
- 无法定位岗位的短链

## 状态枚举

| 状态 | 说明 | 下一步 |
|------|------|--------|
| 待用户确认 | 初始状态 | 等待用户选择 |
| 用户已选择 | 用户确认投递 | 执行投递 |
| 已打开 | 已打开岗位页面 | 填写表单 |
| 已投递 | 投递成功 | 记录结果 |
| 投递受阻 | 遇到障碍 | 处理问题 |
| 需用户登录 | 需要登录 | 等待用户操作 |
| 已跳过 | 用户跳过 | 不处理 |

## 禁止事项

- 未经用户确认自动投递
- 保存、索要或复用用户账号密码
- 编造岗位、公司或链接
- 将广告或培训信息当成招聘岗位

## 投递记录格式

| 字段 | 内容 |
|------|------|
| 岗位 | 岗位名称、公司、平台、详情页 |
| 用户确认 | 用户选择和确认投递的原句或时间 |
| 使用材料 | PDF 简历、HTML 预览、投递说明 |
| 已执行动作 | 打开页面、登录等待、填表、上传、提交 |
| 结果 | 已投递、停在最终确认、需登录、失败 |
| 证据 | 页面标题、时间、错误信息或下一步 |

## 最终提交规则

- 用户已明确授权”提交/投递”时，可以点击最终提交
- 平台提示付费、隐私授权、职位不一致、岗位已下线、材料不可确认时，**不提交**，停在确认点并说明原因
