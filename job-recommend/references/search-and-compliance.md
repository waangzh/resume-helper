# 岗位搜索与合规

## 按岗位类型搜索

### 技术岗搜索策略

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1 | 岗位名 + 类型 | “Java后端 实习” |
| 2 | 加入城市 | “Java后端 实习 北京” |
| 3 | 加入核心技术 | “Java后端 实习 北京 Spring” |
| 4 | 指定平台 | “Java后端 实习 北京 Spring BOSS直聘” |

### 通用岗搜索策略

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1 | 岗位名 + 变体名 | “管培生 OR 管理培训生 OR 储备干部” |
| 2 | 加入类型 + 城市 | “管培生 校招 北京” |
| 3 | 加入行业偏好 | “管培生 校招 北京 互联网” |
| 4 | 指定平台 | “管培生 校招 北京 BOSS直聘” |

**通用岗岗位名称变体：**

| 主岗位名 | 变体名称 |
|----------|----------|
| 管培生 | 管理培训生、储备干部、管培、MT |
| 产品运营 | 运营、用户运营、内容运营、活动运营 |
| 人力资源 | HR、人事、招聘专员、HR助理、HRBP |
| 市场销售 | 销售代表、商务拓展、BD、市场专员 |
| 行政助理 | 行政专员、文员、前台、行政助理 |

## 默认工具：autoglm-websearch

快速关键词搜索，无需登录，响应速度快。

**使用方式：**

直接用自然语言描述任务，agent 会自动调用 autoglm-websearch 执行。

**岗位搜索提示词示例：**

```
使用 autoglm-websearch 搜索 Java后端实习岗位 招聘信息，获取岗位名称、公司、城市、薪资、学历要求和岗位详情页链接

使用 autoglm-websearch 搜索 管培生校招岗位 北京上海 招聘信息，获取前5个岗位的详细信息
```

## 进阶工具：autoglm-browser-agent

当 autoglm-websearch 不可用，或需要完整浏览器操作（如投递表单填写）时使用。

详细文档位于：`~/.openclaw-autoclaw/skills/autoglm-browser-agent/SKILL.md`

### 首次使用准备

1. **安装浏览器扩展**：
   - Chrome：[AutoGLM 扩展](https://chromewebstore.google.com/detail/autoglm/jelniggicmclhfgnlapbkgfibmgelfnp)
   - Edge：[AutoGLM 扩展](https://microsoftedge.microsoft.com/addons/detail/autoglm/ljlnbmmmgnflklegiafalpieckpihffn)

2. **确认配置**：首次使用需确认：
   - 浏览器偏好（Chrome / Edge）
   - 信任模式（敏感操作是否自动执行）

### 使用方式

直接用自然语言描述任务，agent 会自动调用 autoglm-browser-agent 执行。

### 关键规则

| 规则 | 说明 |
|------|------|
| 明确平台 | 指定招聘平台（BOSS直聘、智联招聘等） |
| 明确岗位 | 包含岗位名称和类型（实习/校招/社招） |
| 明确数量 | 默认上限 5 个，用户主动要求更多时除外 |
| 明确字段 | 列出需要收集的信息（名称、公司、城市、薪资、学历、链接等） |
| 可选筛选 | 可指定城市、学历、工作经验等筛选条件 |

### 投递场景示例

```
使用 autoglm-browser-agent 打开岗位页面 https://www.zhipin.com/job_detail/xxx，填写投递表单并上传简历
```

### 遇到登录/验证码

1. 浏览器暂停，返回 `[INTERACT_REQUIRED]`
2. 告知用户手动完成登录
3. 用户确认后继续执行

## 无浏览器工具时

若以上工具均不可用：
1. 输出搜索关键词和筛选规则
2. 提供平台入口链接

## 链接要求

**必须：** 岗位详情页链接，能直接打开到具体岗位

**禁止：**
- 公司官网首页
- 招聘首页
- 搜索结果页
- 聚合页
- 无法定位岗位的短链

## 状态枚举

| 状态 | 说明 | 下一步 |
|------|------|--------|
| 待用户确认 | 初始状态 | 等待用户选择 |
| 用户已选择 | 用户确认投递 | 执行投递 |
| 已打开 | 已打开岗位页面 | 填写表单 |
| 已投递 | 投递成功 | 记录结果 |
| 投递受阻 | 遇到障碍 | 处理问题 |
| 需用户登录 | 需要登录 | 等待用户操作 |
| 已跳过 | 用户跳过 | 不处理 |

## 禁止事项

- 未经用户确认自动投递
- 保存、索要或复用用户账号密码
- 编造岗位、公司或链接
- 将广告或培训信息当成招聘岗位

## 投递记录格式

| 字段 | 内容 |
|------|------|
| 岗位 | 岗位名称、公司、平台、详情页 |
| 用户确认 | 用户选择和确认投递的原句或时间 |
| 使用材料 | PDF 简历、HTML 预览、投递说明 |
| 已执行动作 | 打开页面、登录等待、填表、上传、提交 |
| 结果 | 已投递、停在最终确认、需登录、失败 |
| 证据 | 页面标题、时间、错误信息或下一步 |

## 最终提交规则

- 用户已明确授权”提交/投递”时，可以点击最终提交
- 平台提示付费、隐私授权、职位不一致、岗位已下线、材料不可确认时，**不提交**，停在确认点并说明原因
