# 岗位搜索与合规

## 搜索策略

- 组合关键词：岗位名、实习/校招、城市、核心技术。
- 优先用户指定平台或公开可访问渠道，包括 BOSS 直聘、拉勾、猎聘、智联招聘、前程无忧、牛客、实习僧、LinkedIn、公司 ATS。
- 每条岗位必须有标题、公司、地点、平台、岗位详情页链接和检索日期。
- 链接必须打开到具体岗位详情页，不能是公司官网首页、招聘首页、搜索结果页、聚合页或无法定位岗位的短链。

## 平台注意事项

- BOSS 直聘等平台可能需要登录或限制自动化；遇到登录、验证码、风控时停下让用户处理。
- 公司 ATS 通常更适合精准投递，但仍需核验岗位页标题和公司名称。
- 第三方聚合站只可作为线索来源，最终推荐应尽量落到原始岗位详情页。

## 筛选策略

- 匹配核心技能和求职阶段。
- 城市偏好优先。
- 对学历、经验、到岗时间不匹配的岗位标记风险。

## Excel 清单

- 推荐岗位最终必须能整理为 `.xlsx`，用于用户筛选、确认和跟踪投递。
- Excel 链接列必须是岗位详情页超链接，不允许写官网首页或搜索页。
- 状态列默认“待用户确认”，投递后更新为“已投递”“投递受阻”“需用户登录”等。
- 备注列用于记录过期风险、登录要求、岗位不匹配点或下一步动作。

## 禁止事项

- 未经用户确认自动投递。
- 保存、索要或复用用户账号密码。
- 编造岗位、公司或链接。
- 将搜索结果中的广告或培训信息当成招聘岗位。

## 确认后投递记录

记录格式：

| 字段 | 内容 |
| --- | --- |
| 岗位 | 岗位名称、公司、平台、详情页 |
| 用户确认 | 用户选择和确认投递的原句或时间 |
| 使用材料 | PDF 简历、HTML 预览、投递说明 |
| 已执行动作 | 打开页面、登录等待、填表、上传、提交 |
| 结果 | 已投递、停在最终确认、需登录、失败 |
| 证据 | 页面标题、时间、错误信息或下一步 |

最终提交规则：

- 用户已经明确授权“提交/投递”时，可以点击最终提交。
- 平台提示付费、隐私授权、职位不一致、岗位已下线、材料不可确认时，不提交，改为停在确认点。
