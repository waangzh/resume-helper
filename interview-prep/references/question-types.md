# 面试题类型

## 题目分类

| 类型 | 数量 | 来源 | 说明 |
|------|------|------|------|
| 岗位基础题 | 5 道 | 岗位画像核心技能 | Java 集合、事务、浏览器机制等 |
| 项目追问题 | 5 道 | 简历中的具体项目 | 必须绑定简历项目 |
| 短板补强题 | 3 道 | 匹配诊断中的缺口 | 帮助用户知道面试前该补什么 |

## 题目格式

```markdown
### Q{序号}：{题目}

**考察点：** {本题考察的核心能力}

**答题要点：**
- {要点1：核心概念/原理}
- {要点2：具体实现/方案}
- {要点3：对比/延伸/注意事项}

**可结合项目：** {简历中的具体项目/模块名称}

**避免的夸大表述：** {风险提示，例如"不要说精通"}
```

## 示例

### Q1：HashMap 的底层实现原理是什么？

**考察点：** Java 集合框架、数据结构

**答题要点：**
- JDK 1.8 使用数组 + 链表 + 红黑树
- put 流程：计算 hash → 定位桶 → 处理冲突
- 扩容机制：负载因子 0.75，2 倍扩容

**可结合项目：** 电商项目中的商品缓存

**避免的夸大表述：** 不要说"精通 HashMap 源码"，除非真正读过源码

## 项目追问类型

| 追问类型 | 示例问题 |
|----------|----------|
| 背景追问 | 为什么选择这个技术方案？ |
| 细节追问 | 具体怎么实现的？代码结构是怎样的？ |
| 问题追问 | 遇到了什么问题？怎么解决的？ |
| 延伸追问 | 如果数据量翻倍，你会怎么优化？ |
