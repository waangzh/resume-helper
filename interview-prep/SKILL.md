# Interview Prep

## 流程

1. 读取岗位画像、最终简历和匹配诊断。
2. 根据 `JobProfile.role_category` 选择题型：
   - 技术岗：岗位基础题、项目追问题、短板补强题
   - 通用岗：行为面试题、情景题、经历深挖题
3. 每题附考察点和答题要点；项目/经历题必须绑定简历中的具体内容。
4. 对风险项生成追问，帮助用户提前补证据或收敛表述。

## 默认输出

### 技术岗

| 类型 | 数量 | 说明 |
|------|------|------|
| 岗位基础题 | 5 道 | 围绕岗位核心技能 |
| 项目追问题 | 5 道 | 绑定简历中的具体项目 |
| 短板补强题 | 3 道 | 围绕匹配诊断中的真实缺口 |
| 面试准备建议 | 1 段 | 优先级排序 |

### 通用岗

| 类型 | 数量 | 说明 |
|------|------|------|
| 行为面试题 | 5 道 | 围绕核心能力（沟通、协作、领导力等） |
| 经历深挖题 | 5 道 | 绑定简历中的具体经历 |
| 情景题 | 3 道 | 模拟工作场景，考察问题解决能力 |
| 面试准备建议 | 1 段 | 优先级排序 |

## 题目格式

### 技术岗

```markdown
### Q{序号}：{题目}

**考察点：** {考察内容}

**答题要点：**
- {要点1}
- {要点2}
- {要点3}

**可结合项目：** {简历中的具体项目/模块}

**避免的夸大表述：** {风险提示}
```

### 通用岗

#### 行为面试题

```markdown
### Q{序号}：{题目}

**考察能力：** {沟通表达/团队协作/领导力等}

**答题要点（STAR）：**
- 情景（S）：{什么情况下}
- 任务（T）：{要解决什么问题}
- 行动（A）：{做了什么}
- 结果（R）：{成果是什么}

**可结合经历：** {简历中的具体经历}

**避免的空泛表述：** {风险提示}
```

#### 情景题

```markdown
### Q{序号}：{题目}

**考察能力：** {问题解决/抗压能力/沟通协调等}

**答题要点：**
- 分析问题：{如何理解问题}
- 提出方案：{如何解决}
- 执行落地：{如何推进}
- 结果复盘：{如何总结}

**评分标准：** {好/中/差的表现}
```

## 行为面试题示例

### 沟通表达

```markdown
### Q1：请描述一次你通过沟通解决冲突的经历

**考察能力：** 沟通表达、冲突处理

**答题要点（STAR）：**
- S：团队内部对活动方案有分歧
- T：需要协调双方达成一致
- A：分别了解双方诉求，找出共同点，提出折中方案
- R：双方接受方案，活动顺利举办

**可结合经历：** 学生会活动组织经历

**避免的空泛表述：** “我善于沟通”、”我协调能力强”（需具体事例）
```

### 团队协作

```markdown
### Q2：请举例说明你在团队中如何支持他人

**考察能力：** 团队协作、服务意识

**答题要点（STAR）：**
- S：队友负责的模块进度落后
- T：需要帮助队友赶上进度
- A：主动分担部分工作，协助梳理思路
- R：团队按期完成任务，队友能力提升

**可结合经历：** 课程项目/社团工作经历
```

### 领导力

```markdown
### Q3：请描述一次你带领团队完成目标的经历

**考察能力：** 领导力、目标达成

**答题要点（STAR）：**
- S：负责XX活动/项目
- T：需要在XX时间内完成XX目标
- A：制定计划、分配任务、跟进进度、解决困难
- R：团队完成目标，获XX认可/荣誉

**可结合经历：** 学生会主席/项目负责人经历
```

## 边界

- 不堆砌题库大全
- 不生成与简历无关的项目/经历追问
- 不把用户不会的短板包装成已掌握能力
- 行为面试题必须有 STAR 答题框架
